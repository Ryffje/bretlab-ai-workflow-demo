bretlab-ai-workflow-demo

Demo end-to-end de un workflow en n8n que recibe mensajes vía webhook (testeado con Postman), normaliza el payload, usa un LLM con salida JSON estructurada para decidir la intención (intent/next_step), enruta con un Switch, consulta/persiste en Supabase y publica la respuesta final a un webhook (webhook.site).

Objetivo

Mostrar un flujo que toma un mensaje entrante, genera una decisión con LLM (output estructurado), ejecuta lógica de ruteo y consulta datos para devolver una respuesta final lista para enviarse al canal.

Seguridad

Tokens/headers y data sensible se ocultan. Este repo es demostrativo: usa .env.example para documentar variables sin subir secretos.

Screenshots (paso a paso)

Secuencia del demo: vista general → trigger → configuración LLM → output estructurado → router → Supabase insert → Supabase fetch con filtro → formateo → respuesta final.

01 — Architecture / Workflow overview Vista general del workflow y componentes principales.
02 — Webhook trigger (Postman) Entrada simulada para pruebas end-to-end (sin proveedor de mensajería).
03 — LLM node config (JSON output) El modelo devuelve JSON válido con claves esperadas (intent, next_step, etc.).
04 — LLM structured output Salida estructurada que alimenta el ruteo (intent/next_step + datos normalizados).
05 — Router / Switch Enrutamiento por next_step (ramas: greeting, fetch_workers, ask_location, ask_service, send_message).
06 — Supabase insert (messages) Persistencia del historial: POST a Supabase REST.
07 — Supabase fetch (workers) Consulta con filtros dinámicos (ej. ilike) según skill/location.
08 — Format response (JS) Formateo final de la lista y fallback si no hay resultados.
09 — Final response (webhook.site) POST final del mensaje listo para enviarse al usuario (demostración completa end-to-end).
